# flash-read1 PRD

## 1. 项目背景

`flash-read1` 当前已具备书架、卷页翻页和本地轻量听书骨架，但阅读页仍存在两个直接影响体验的问题：语音预设容易命中同一系统 voice，用户感知为“只有一个语音包能用”；翻页提示文案出现在纸张正文区域，侵占了可读内容空间。本轮在不扩大产品范围的前提下，收敛为一次面向阅读主链路的小范围修复与体验微调。

## 2. 本轮目标

- 修复语音预设映射策略，让多个预设在支持环境下尽量可区分
- 在设备 voice 有限时，为语音回退提供明确反馈，避免用户误判为功能失效
- 将翻页操作提示从正文纸张中移出，恢复正文区的阅读优先级
- 轻量提升阅读页的内容密度和沉浸感，观察“每页更多正文”后的显示效果
- 保持现有书架、翻页手势、朗读控制和阅读进度行为不回归

## 3. 目标用户

- 碎片化阅读的移动端用户
- 需要边读边听、会主动切换语音预设的用户
- 用当前阅读器骨架验证交互和视觉密度的内部研发与设计团队

## 4. 使用场景

### 场景 A：切换语音预设后继续听书

1. 用户在阅读页打开语音预设列表
2. 用户切换到另一种语音风格并开始朗读
3. 系统优先匹配与当前预设更接近的可用 voice
4. 若设备可用 voice 不足，界面明确说明已回退为默认或近似 voice

### 场景 B：阅读正文时不被翻页文案干扰

1. 用户进入阅读页查看纸张正文
2. 纸张中部只展示章节内容，不显示“轻触上一页/下一页”等辅助操作文案
3. 用户仍可通过左右边缘热区或滑动手势完成翻页

### 场景 C：单页承载更多内容

1. 用户在阅读页持续浏览多页内容
2. 单页正文显示量较上一版更高，减少“内容被提示占位”和过早翻页的感觉
3. 顶部返回、页码、语音控制仍然清晰可用

### 场景 D：保持既有主流程稳定

1. 用户从书架进入阅读页并翻页
2. 用户播放、暂停、停止当前页朗读
3. 用户翻页、切换语音或返回书架时，上一段朗读按既有逻辑停止

## 5. MVP 范围

### 5.1 本轮纳入

- 书架到阅读页的现有主流程保持不变
- 阅读页继续支持左右滑动翻页和边缘点击翻页
- 语音预设映射策略优化为更可区分的匹配逻辑
- 语音未命中或设备受限时展示轻量回退提示
- 翻页提示迁移到纸张外或边缘辅助区
- 阅读页做轻量视觉层级和内容密度微调
- mock 书页内容适度扩充，用于验证显示效果

### 5.2 本轮不纳入

- 新的 TTS 服务、音频缓存、离线包或云端语音
- 阅读器整体重构、翻页动画重做、导航重构
- 目录、搜索、持久化、主题系统、排版引擎
- 真实书源接入、登录、同步、评论与批注能力

## 6. 验收标准

### 6.1 功能验收

- App 能正常启动到书架页，并进入阅读页
- 阅读页可以继续完成滑动翻页与边缘快速翻页
- 至少两个语音预设在支持多 voice 的设备或模拟环境下可命中不同 voice
- 设备 voice 有限或未命中预设时，界面给出默认或近似 voice 的回退说明
- 切换语音预设后再开始朗读，仍只朗读当前页
- 翻页、切换语音预设、关闭阅读页时，当前朗读会先停止

### 6.2 体验验收

- 纸张正文区域不再出现翻页操作文案
- 单页正文展示量高于当前上一版本，且不遮挡顶部返回和底部听书控制
- 操作层级符合阅读优先，正文、副信息和操作提示一眼可区分
- 翻页热区仍可触达，且不明显侵占正文滚动区域

## 7. 非功能要求

- 保持 TypeScript 类型与现有 Context/reducer 状态模型一致
- 维持 Expo Managed Workflow，不增加高风险依赖
- 文档与实现边界一致，不引入未计划的新功能描述
- 本轮至少通过 `npm run typecheck`

## 8. 边界条件与假设

- 当前版本仅使用本地 mock 数据，不接入真实排版引擎
- `expo-speech` 的可用 voice 由设备和平台决定，无法保证所有预设都有独立音色
- 语音问题假设主要源于预设匹配过宽或多个预设命中同一 voice，而非底层 TTS API 不可用
- 内容密度提升只做轻量调整，不追求复杂排版、字号配置或阅读主题切换

## 9. 风险与回退

- 风险：部分设备 voice 数量过少，仍可能无法让所有预设明显区分
- 风险：正文区域增加内容后，边缘热区与滚动可用性可能出现新的误触
- 风险：UI 微调如果过度压缩留白，可能损伤可读性
- 回退：若未能可靠区分全部预设，至少保留回退提示并避免“只有一个能用”的错误感知
- 回退：若纸张外提示影响手势或布局，可缩减为更弱的边缘辅助标识，不恢复到正文区

## 10. 交付结论

本轮交付是一次限定在阅读页主链路内的增量迭代，目标是同时完成语音映射修复、翻页提示布局修正，以及阅读页 UI 与 mock 内容的轻量优化，并确保范围、风险和验收标准可供 coder 与 tester 独立执行。
