# flash-read1 PRD

## 1. 项目背景

`flash-read1` 当前已具备书架、卷页翻页和本地轻量听书骨架，但阅读页仍存在两个直接影响体验的问题：语音预设容易命中同一系统 voice，用户感知为“只有一个语音包能用”；翻页提示文案出现在纸张正文区域，侵占了可读内容空间。本轮在不扩大产品范围的前提下，收敛为一次面向阅读主链路的小范围修复与体验微调。

## 2. 本轮目标

- 去掉卷页翻动特效，只保留底页由淡到清晰的翻页反馈
- 收敛书架 mock 数据，只保留一本用于功能联调的 Node.js 主书
- 将 `Node.js Design Patterns` 扩充为覆盖主要章节的完整版摘要页，避免继续停留在三页试读
- 保持现有阅读页翻页手势、朗读控制和阅读进度行为不回归

## 3. 目标用户

- 碎片化阅读的移动端用户
- 需要边读边听、会主动切换语音预设的用户
- 用当前阅读器骨架验证交互和视觉密度的内部研发与设计团队

## 4. 使用场景

### 场景 A：切换语音预设后继续听书

1. 用户在阅读页打开语音预设列表
2. 用户切换到另一种语音风格并开始朗读
3. 系统优先匹配与当前预设更接近的可用 voice
4. 若设备可用 voice 不足，界面明确说明已回退为默认或近似 voice

### 场景 B：阅读正文时不被翻页文案干扰

1. 用户进入阅读页查看纸张正文
2. 纸张中部只展示章节内容，不显示“轻触上一页/下一页”等辅助操作文案
3. 用户仍可通过左右边缘热区或滑动手势完成翻页

### 场景 C：单页承载更多内容

1. 用户在阅读页持续浏览多页内容
2. 单页正文显示量较上一版更高，减少“内容被提示占位”和过早翻页的感觉
3. 顶部返回、页码、语音控制仍然清晰可用

### 场景 D：Node.js 主书作为唯一功能联调书目

1. 用户进入书架后只看到 `Node.js Design Patterns`
2. 用户打开后可连续翻阅完整章节摘要，而不是三页试读内容
3. 研发团队后续围绕同一本书继续验证阅读器功能

### 场景 D：保持既有主流程稳定

1. 用户从书架进入阅读页并翻页
2. 用户播放、暂停、停止当前页朗读
3. 用户翻页、切换语音或返回书架时，上一段朗读按既有逻辑停止

## 5. MVP 范围

### 5.1 本轮纳入

- 书架到阅读页的现有主流程保持不变
- 阅读页继续支持左右滑动翻页和边缘点击翻页
- 阅读页翻页反馈收敛为前景页平移 + 底页渐清晰，不再保留卷页特效
- 书架 mock 数据收敛为单一本地 Node.js 主书
- `Node.js Design Patterns` 扩展为覆盖主要章节的长页摘要，直接用于后续功能验证

### 5.2 本轮不纳入

- 新的 TTS 服务、音频缓存、离线包或云端语音
- 阅读器整体重构、导航重构
- 目录、搜索、持久化、主题系统、排版引擎
- 真实书源接入、登录、同步、评论与批注能力

## 6. 验收标准

### 6.1 功能验收

- App 能正常启动到书架页，并进入阅读页
- 书架当前仅保留一本 `Node.js Design Patterns`
- 阅读页可以继续完成滑动翻页与边缘快速翻页
- 阅读页翻页时不再出现卷页旋转、卷角或书脊阴影特效
- 底页内容仍可在翻页过程中由淡到清晰地显露
- `Node.js Design Patterns` 至少覆盖主要章节，不再只有三页
- 切换语音预设后再开始朗读，仍只朗读当前页
- 翻页、切换语音预设、关闭阅读页时，当前朗读会先停止

### 6.2 体验验收

- 单页正文展示量高于当前上一版本，且不遮挡顶部返回和底部听书控制
- 操作层级符合阅读优先，正文、副信息和操作提示一眼可区分
- 翻页热区仍可触达，且不明显侵占正文滚动区域
- 翻页视觉反馈更克制，不再有影响阅读沉浸感的卷页表演效果

## 7. 非功能要求

- 保持 TypeScript 类型与现有 Context/reducer 状态模型一致
- 维持 Expo Managed Workflow，不增加高风险依赖
- 文档与实现边界一致，不引入未计划的新功能描述
- 本轮至少通过 `npm run typecheck`

## 8. 边界条件与假设

- 当前版本仅使用本地 mock 数据，不接入真实排版引擎或真实书源
- `expo-speech` 的可用 voice 由设备和平台决定，无法保证所有预设都有独立音色
- 当前 `Node.js Design Patterns` 内容为按章节整理的摘要版 mock，而非原书全文
- 内容密度提升只做轻量调整，不追求复杂排版、字号配置或阅读主题切换

## 9. 风险与回退

- 风险：正文区域增加内容后，边缘热区与滚动可用性可能出现新的误触
- 风险：单一本地书目会让书架暂时失去对多书场景的覆盖
- 回退：若翻页表演层去掉后手感过硬，优先微调位移时长和底页透明度，不恢复卷页旋转与卷角
- 回退：若章节摘要仍不足以支撑联调，再继续扩充当前 Node.js 主书页数，不恢复其他 mock 书目

## 10. 交付结论

本轮交付收敛为阅读器主链路的继续打磨，重点是拿掉卷页表演层、保留底页渐显反馈，并把书架数据压缩到一本可持续联调的 Node.js 主书。
